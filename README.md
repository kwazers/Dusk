# trapbox
Кроссплатформенный веб-обозреватель с модульной архитектурой и повышенным уровнем конфиденциальности.

Проект представляет собой кастомный браузер, построенный на базе среды выполнения Node.js и движка Chromium (с использованием фреймворка Electron.js). В отличие от стандартных браузеров, данное решение ориентировано на полную изоляцию пользовательских данных, встроенную анонимизацию трафика и эффективное управление системными ресурсами.

#🛠 Ключевая Архитектура и Технологии
Проект реализует многопроцессную архитектуру, разделяющую логику управления и рендеринг веб-страниц:

- Main Process (Node.js бэкенд): Отвечает за системные функции, управление сессиями, запуск сторонних бинарных процессов (Tor), маршрутизацию трафика и работу с файловой системой ПК.

- Renderer Process (Фронтенд): Интерфейс браузера (боковая панель, вкладки, менеджеры) полностью построен на стеке HTML5, CSS3 (Flexbox/Grid) и JavaScript (ES6).

- Изоляция контекста: Безопасность системы обеспечивается за счет принудительного включения contextIsolation и отключения nodeIntegration для гостевых веб-страниц, что исключает выполнение вредоносного кода на хост-машине.

#🚀 Основной Функционал (Модули и Плагины)
Многопрофильный Антидетект (Контейнеризация):
Реализована изоляция куки-файлов, кэша и локальных хранилищ для каждой отдельной вкладки или группы вкладок с помощью изолированных сессий (session.fromPartition). Позволяет авторизоваться в разных аккаунтах одного сервиса в соседних вкладках без пересечения данных. Поддерживается динамическая подмена User-Agent

- Интеграция Сети Tor:
Маршрутизация сетевого трафика через оверлейную сеть Tor на уровне бэкенда (setProxy). Браузер позволяет гибко пускать через Tor как весь трафик, так и строго определенные приватные сессии, сохраняя максимальную скорость на остальных вкладках.

- Оптимизация ОЗУ (Tab Suspender):
Нативный алгоритм мониторинга активности вкладок. При долгом отсутствии взаимодействия вкладка «замораживается» (выгружается из оперативной памяти с уничтожением инстанса страницы), сохраняя свое визуальное состояние в боковой панели. При возврате пользователя происходит мгновенный пре-рендер.

- Нативный Фильтр Сетевых Запросов (AdBlock):
Блокировка рекламы, трекеров и аналитических скриптов осуществляется на уровне перехвата сетевых запросов (webRequest.onBeforeRequest) до этапа загрузки элементов на страницу, что ускоряет рендеринг сайтов в 2–3 раза.

- Встроенная Боковая Панель (Сайдбар):
Вертикальный эргономичный интерфейс для быстрого переключения между изолированными профилями, вкладками и встроенными утилитами.

- Поддержка Расширений Chromium:
Интеграция прослойки для эмуляции API Chrome-расширений, позволяющая устанавливать любые распакованные плагины (например, криптокошельки, менеджеры паролей) напрямую из Chrome Web Store.

- Локальный Кастомизатор:
Модуль управления интерфейсом, позволяющий сохранять локальные конфигурации (кастомные обои, темы оформления) в формате JSON.
